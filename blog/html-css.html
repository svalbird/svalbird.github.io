CSS: Getting in the right position

18 January 2022

Positioning CSS elements can be a major headache. The browser has ideas about where things should go, and… well, it’s trying it’s best. Often the best course of action is to just tell it where to put things on the webpage. But there’s a lot of ways of doing that, and picking the right one at the right time can save you from finding fun formatting surprises later on in designing your page. The default position for an element is static. That just means “it goes where it goes”. That’s boring, so lets skip to the interesting ones.

Getting relative

The easiest type of element positioning to conceptualize is relative positioning. Simply type in

position: relative;

And watch with amazement as nothing happens! Well, nothing should happen, at least. Relative positioning is simply telling the browser “Keep it where it is, I’ll move it around from there.”. With that established we can use that position as a baseline and go from there. For example

right: 30px; top: 20px;

would move that element 30 pixels away from the right and 20px away from the top from where it already was. Keep in mind that you’re defining the distance the element is from a particular direction, and so it’ll move away from what you specify. Relative positioning is useful when you need to adjust things slightly or when the rough formatting is there, but the precise location needs to be fiddled with. It also maintains its position relative to other elements in the same container, so it’s a bit neater when there’s moving parts or you might need to resize some stuff. That can be a hindrance if you find yourself having to mess with it when you change something, however.

Absolute uncertainty

Sometimes the browser gets it completely wrong. That’s ok – its what absolute positioning is for! Absolute positioning allows you to tell the browser exactly where the element goes in precise terms in its container. For example

position: absolute; left:40px; top: 50px;

would move the element to exactly 40px from the left and 50px from the top of the containing block it’s in. It’ll ignore whatever formatting was already there. This type of positioning is great for doing unorthodox things or for making sure things stay in a specific spot. It’s inflexible, however, and as making an element’s position absolute can force ti to be ignored by layouts, sometimes elements that would contain it break and need to be adjusted to compensate.

Fixed (for now)

Fixed position is like absolute position but more. It’s the same, with precise information on its location needed, but it’s always relative to the initial viewport (that is, that entire webpage). For example

position: fixed; left:40px; top: 50px;

would put an element 40px from the left and 50px from the top of the entire page. This is great for ensuring something is exactly where you want it, but this brute-force method has a clear downside – it’s not going to adjust automatically when you change things around it. This is important to keep in mind.

This list isn’t comprehensive on every aspect of positioning in CSS – looking further into the specific of each is recommended, especially if you’re doing something odd with positions, as they have some specific interactions with things such as inheritance and tables in some cases that can be a bit confusing. Overall, positions are critical in effective CSS coding. Understanding and mastering different types, when to use then, and when not to use them, are skills that will benefit any web designer massively.
